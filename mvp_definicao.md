# MVP (Produto Mínimo Viável) - Plataforma de Comunicação e Feedback

## Objetivo do MVP:
Oferecer um canal centralizado e seguro para comunicação interna e feedback, permitindo aos colaboradores se sentirem informados e ouvidos, enquanto o RH/gestores podem comunicar e coletar feedback de forma eficaz e transparente. O foco é resolver as "dores nº 1" da persona Ana Clara: a fragmentação da comunicação e o medo de retaliação ao dar feedback.

## Hipótese a ser Testada:
"Se fornecermos aos colaboradores um aplicativo mobile simples para receber comunicados oficiais e um canal genuinamente anônimo para dar feedback, eles se sentirão mais informados e seguros para expressar suas opiniões, aumentando o engajamento e a transparência na empresa."

## Funcionalidades Essenciais do MVP:

### 1. Aplicativo Mobile para Colaboradores (Mínimo Viável)
*   **Autenticação de Usuário:**
    *   Login básico com e-mail corporativo e senha.
*   **Feed de Comunicados:**
    *   Exibição de uma lista cronológica simples de comunicados oficiais (apenas texto e imagens básicas).
    *   Clicar em um comunicado exibe seu conteúdo completo.
*   **Confirmação de Leitura ("Ciente"):**
    *   Botão "Li e Concordo" visível em comunicados que o RH/Gestor marcar como obrigatórios.
    *   Registro interno de que o colaborador confirmou a leitura.
*   **Canal de Feedback/Sugestões/Denúncias:**
    *   Formulário simplificado (apenas campo de texto livre) para envio de feedback.
    *   Opção clara e funcional para enviar o feedback de forma anônima.

### 2. Portal Web para Gestores de RH / Líderes de Equipe (Mínimo Viável)
*   **Autenticação de Usuário:**
    *   Login básico para gestores de RH/líderes de equipe.
*   **Gerenciamento de Comunicados:**
    *   Interface para criação e publicação de novos comunicados (título, corpo de texto simples).
    *   Opção para marcar um comunicado como "requer ciente".
    *   Listagem simples de todos os comunicados publicados.
*   **Visualização de Feedbacks:**
    *   Interface para visualizar uma lista de feedbacks recebidos.
    *   Indicação clara se o feedback é anônimo ou identificado.

## Funcionalidades para Futuras Iterações (não no MVP):

*   **Para o Colaborador:**
    *   Filtros e busca avançada no feed de comunicados.
    *   Marcação de comunicados como "favoritos".
    *   Histórico detalhado de interações (comunicados lidos, feedbacks enviados, status).
    *   Notificações push personalizáveis.
    *   Upload de arquivos/imagens nos feedbacks.
    *   Acompanhamento do status do feedback pelo colaborador.
*   **Para o Gestor/RH:**
    *   Edição de comunicados após publicação.
    *   Gerenciamento de usuários e permissões.
    *   Relatórios detalhados sobre "cientes" e volume/tipo de feedbacks.
    *   Definição de públicos-alvo específicos para comunicados.
    *   Ferramentas para categorizar e gerenciar o status dos feedbacks (além de apenas visualizar).
    *   Funcionalidades multi-tenant para consultorias.
    *   Análise de sentimento via IA.
    *   Dashboards e KPIs avançados.
    *   Integração com sistemas externos (HRIS, folha de pagamento).
    *   Módulos adicionais (avaliação de desempenho, enquetes, gamificação).

Este MVP foca na entrega do valor fundamental e no aprendizado inicial, minimizando o esforço e o tempo de desenvolvimento para validar as premissas mais importantes do projeto.
